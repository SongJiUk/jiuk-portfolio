01데이터 기반 콘텐츠 및 캐릭터 성장 관리
PROBLEM

게임 밸런스 조정 및 콘텐츠 추가 시 코드 수정이 필요하여 개발 효율이 떨어졌습니다.

SOLUTION

모든 게임 데이터를 구글시트에서 관리하여 개발환경과 분리, 코드를 수정하지 않고도 게임의 밸런스와 콘텐츠를 유연하게 조정할 수 있는 데이터 기반 설계를 적용했습니다.

몬스터·스킬·캐릭터·장비 데이터를 구글 시트에 정리 → CSV → JSON으로 변환 → JsonUtility로 C# 객체로 변환. ILoader 인터페이스로 List를 Dictionary로 변환해 O(1) 탐색을 최적화했습니다.

구글시트에서 데이터 관리

구글시트 → Json파일로 변환

Json → C# 객체로 변환 (Newtonsoft.Json)

ILoader로 List → Dictionary 변환, O(1) 조회 최적화

플레이어 성장 시스템: 장비 합성, 정렬, 강화 / 캐릭터 해방 / 뽑기 시스템 / 출석체크 & 미션

02핵심 게임 로직 및 최적화
PROBLEM

수많은 몬스터와 오브젝트가 동시다발적으로 생성·소멸되는 환경에서 메모리 할당/해제 오버헤드로 인한 성능 저하가 있었습니다.

SOLUTION

Unity의 ObjectPool<T>를 적극 활용하여 메모리와 CPU 부하를 최소화했습니다.

몬스터를 처치하여 얻은 젬으로 레벨업하고, 무작위로 등장하는 스킬을 선택하여 성장하는 로그라이크 플레이를 구현했습니다. 스킬 레벨이 5가 된 후 특정 아이템을 획득하면 스킬이 진화하는 시스템으로 전략적 깊이를 더했습니다.

로그라이크 스킬 선택 / 스킬 레벨 5 진화 시스템

10개 스테이지 / 엘리트 보스 / 자기장 시스템으로 긴장감 조성

ObjectPool — OnCreate, OnGet, OnRelease, OnDestroy 구현

오브젝트 풀을 관리하는 코드

실제 오브젝트 풀을 사용하는 코드

03중앙 집중식 업데이트 시스템
PROBLEM

유니티의 개별적인 Update() 호출 방식은 오브젝트 수가 많아질 때 불필요한 호출 오버헤드를 유발하여 성능 저하로 이어집니다.

SOLUTION

ITickable 인터페이스와 UpdateManager를 구축하여 모든 업데이트 로직을 일괄적으로 관리했습니다.

UpdateManager — 연결된 오브젝트들의 업데이트 로직을 일괄 관리

필요한 스크립트에서 Register/Unregister로 관리

04Drag & Drop 없는 UI 바인딩 시스템
PROBLEM

기존 Unity UI 작업 방식은 수동 연결로 인한 반복 작업, 휴먼에러, 유지보수의 어려움이 있었습니다.

SOLUTION

Enum과 코드 기반의 자동 UI 바인딩 시스템을 구축했습니다. 코드로 컴포넌트 간 의존성을 명확하게 관리하고, UI의 유지보수성 및 확장성을 확보했습니다.

Enum 기반으로 컴포넌트를 찾아 자동 연결하는 Bind<T>() 코드

프리팹에서 컴포넌트 이름과 Enum 이름을 일치시켜 자동 연결

Enum 기반으로 컴포넌트를 찾아 연결하는 코드

특정 오브젝트에 이벤트를 바인드하는 코드

어디서든 쉽게 사용할 수 있도록 Utils로 만든 코드

EventHandler를 이용해 게임오브젝트에 이벤트를 바인드하는 코드
