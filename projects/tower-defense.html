01A* 경로탐색 + 동적 재탐색

스테이지 타일맵을 기준으로 GridSystem이 격자를 생성하고, 각 타일을 GridNode로 등록합니다. Road 레이어 타일은 이동 가능(CanWalk = true), 그 외는 벽으로 처리해 A*가 탐색할 그래프를 구성합니다.

A* 알고리즘을 직접 구현. 스킬 장벽(Block)으로 경로 위에 상자를 설치해 적의 이동 경로를 능동적으로 조작할 수 있습니다. 상자 설치 시 RecalculatePath() 이벤트로 모든 적이 현재 위치에서 즉시 경로를 재계산합니다. 단, 모든 경로를 완전히 막는 것은 불가능하도록 설계했습니다 — 장벽의 지속시간이 쿨타임보다 짧아 상자가 사라지기 전에 다음 장벽을 설치할 수 없으므로, 경로를 영구적으로 봉쇄할 수 없습니다. 맨해튼 거리 휴리스틱, 탐색 후 G/H/Parent 초기화로 다음 호출 오염을 방지합니다.

C#PathFinder.cs
public List<Vector3> FindPath(Vector3 startPos, Vector3 endPos)
{
    GridNode startNode = Managers.Grid.GetNode(startPos);
    GridNode endNode   = Managers.Grid.GetNode(endPos);

    var open    = new List<GridNode>();
    var closed  = new HashSet<GridNode>();
    var visited = new List<GridNode>();

    startNode.G = 0;
    startNode.H = GetHeuristic(startNode, endNode); // 맨해튼 거리
    open.Add(startNode);

    while (open.Count > 0)
    {
        GridNode current = GetLowestF(open);
        if (current == endNode)
        {
            List<Vector3> path = TracePath(endNode);
            ResetNodes(visited); // G·H·Parent 초기화
            LastPath = path;
            return path;
        }
        open.Remove(current);
        closed.Add(current);

        foreach (var (dx, dz) in _directions) // 상하좌우 4방향
        {
            GridNode nb = grid.GetNode(current.GridX + dx, current.GridZ + dz);
            if (nb == null || !nb.CanWalk || closed.Contains(nb)) continue;
            int newG = current.G + 1;
            if (!open.Contains(nb)) { nb.G = newG; nb.H = GetHeuristic(nb, endNode); open.Add(nb); }
            else if (newG < nb.G) nb.G = newG;
        }
    }
    return null; // 경로 없음 → 타워 설치 취소
}
02버프/디버프 시스템 (BuffHandler)

IBuff 인터페이스 + BuffHandler 컴포넌트로 슬로우·독·동결·공격강화를 다중 스택 관리합니다. AllowStack 플래그로 중복 시 Refresh/스택 분기를 결정하고, StatModifier 패턴으로 여러 버프가 중첩된 최종 스탯을 연산합니다. ITickable로 등록해 별도 코루틴 없이 매 프레임 만료를 체크합니다.

C#BuffHandler.cs
public void AddEffect(BuffEffect effect)
{
    if (!effect.AllowStack)
    {
        foreach (var e in _effects)
        {
            if (e is BuffEffect existing && existing.EffectType == effect.EffectType)
            { existing.Refresh(); return; } // 중복 시 지속시간만 갱신
        }
    }
    _effects.Add(effect);
    effect.OnApply(this);
}

// ITickable.Tick: 만료 효과를 버퍼에 모아 한 번에 제거 (순회 중 컬렉션 수정 방지)
public void Tick(float dt)
{
    foreach (var e in _effects)
    { e.Tick(dt); if (e.IsExpired) _expiredBuffer.Add(e); }

    foreach (var e in _expiredBuffer)
    { e.OnRemove(this); _effects.Remove(e); }
    _expiredBuffer.Clear();
}

// StatModifier로 최종 스탯 계산 (배율·덧셈 조합)
public float GetStat(Define.StatType statType, float baseValue)
{
    foreach (var m in _modifiers)
        if (m.StatType == statType) baseValue = m.Apply(baseValue);
    return baseValue;
}
03EffectManager - Pool 기반 이펙트 일원화

투사체 착탄·사망·버프·스킬·타워 설치 등 모든 이펙트를 EffectManager 한 곳으로 통일했습니다. Play(key, pos) 한 줄로 이펙트를 재생하고, async void 대신 UniTaskVoid를 사용해 예외 안전성을 확보합니다. duration 후 자동으로 풀에 반환되므로 호출부에서 별도 처리가 필요 없습니다.

C#EffectManager.cs
public void Play(string key, Vector3 pos, float duration = 2f)
{
    GameObject go = Managers.PoolM.Pop(key);
    if (go == null) return;
    go.transform.position = pos;
    ReturnAsync(go, duration).Forget();
}

private async UniTaskVoid ReturnAsync(GameObject go, float duration)
{
    await UniTask.Delay(TimeSpan.FromSeconds(duration));
    if (go != null && go.activeSelf) Managers.PoolM.Push(go);
}
04타워 6종 + 시너지 시스템

Basic·Cannon·Slow·Sniper·Poison·Lightning 각각 독립 Controller로 구현했습니다. 타워를 배치하면 SynergyManager가 조합을 감지해 OnSynergyChanged 이벤트를 발행하고, 모든 타워가 구독해 스탯을 자동 재계산합니다. 카드 효과도 OnCardApplied 이벤트로 동일하게 반영됩니다. 업그레이드 수치는 ScriptableObject에 분리해 코드 변경 없이 밸런싱이 가능합니다.

시너지조합효과
⚡☠️ 전도성 독독 + 번개독 걸린 적에게 체인 데미지 ×1.5
🎯☠️ 독침독 + 저격독 걸린 적 저격 시 크리티컬 확률 +25%
🐢⚡ 도체슬로우 + 번개슬로우 걸린 적 명중 시 체인 1회 추가
🐢💣 정밀 포격슬로우 + 캐논슬로우 걸린 적 명중 시 스플래시 반경 ×1.5
🔫🎯 집중사격기본 + 저격두 타워 모두 공격속도 +15%
C#TowerController.cs
void OnEnable()
{
    // 카드 효과·시너지 변경 시 스탯 자동 재계산
    Managers.GameM.OnCardApplied      += ApplyStats;
    Managers.SynergyM.OnSynergyChanged += ApplyStats;
}

void OnDisable()
{
    Managers.GameM.OnCardApplied      -= ApplyStats;
    Managers.SynergyM.OnSynergyChanged -= ApplyStats;
    Managers.SynergyM.Unregister(Data.towerType);
}
05Managers 싱글톤 아키텍처

@Managers GameObject를 자동 생성 후 DontDestroyOnLoad로 유지합니다. Pool·Resource·Effect·UI·Game·Card·Skill·Synergy·Sound·Save·Firebase·Difficulty 등 15개 서브매니저를 모듈화해 씬 어디서든 Managers.PoolM, Managers.SkillM으로 접근합니다. 씬 전환 시 Managers.Clear()로 Pool·Resource·UI·Synergy를 일괄 초기화합니다.

C#Managers.cs
public class Managers : MonoBehaviour
{
    readonly PoolManager       poolManager       = new();
    readonly EffectManager     effectManager     = new();
    readonly SkillManager      skillManager      = new();
    readonly SynergyManager    synergyManager    = new();
    readonly DifficultyManager  difficultyManager = new();
    readonly SaveManager       saveManager       = new();
    // ... 15개 서브매니저

    void OnApplicationPause(bool pause)
    {
        if (pause) SaveM?.SyncToFirebase(); // 백그라운드 진입 시 자동 저장
    }

    public static void Clear()
    {
        PoolM.Clear(); ResourceM.Clear();
        UIM.Clear();  SynergyM.Clear();
    }
}
06로그라이크 카드 · 스킬 시스템

웨이브 클리어 후 카드 3장 중 1장을 선택합니다.
카드는 4개 카테고리로 구성됩니다: A(전투강화) — 공격력·공속·사거리·치명타 강화 / B(경제) — 즉시 골드·처치 보상 증가·건설비 감소·웨이브 보너스 2배 / C(특수) — 무료 타워·적 HP 감소·코어 HP 회복·시너지 증폭 / D(스킬) — 스킬 선택·스킬 포인트 추가.
스킬은 5종, 마스터 레벨 3, 최대 3개 보유·교체 가능. 교체 시 레벨 초기화. 모든 데이터는 ScriptableObject로 분리해 하드코딩이 없습니다.

🧱 장벽 (Block)
LV 1→3
경로 위에 상자 설치. 적의 이동 경로를 능동적으로 우회시킴. 지속시간 < 쿨타임으로 영구 봉쇄 불가.
🏹 화살 소나기 (ArrowRain)
LV 1→3
지정 범위에 화살 다수를 쏟아부어 광역 피해. 범위와 화살 수가 레벨에 따라 증가.
❄️ 동결 (Freeze)
LV 1→3
범위 내 적을 일정 시간 완전 정지. 레벨이 높을수록 지속시간과 범위 증가.
⚡ 번개 폭풍 (LightningStorm)
LV 1→3
광역에 번개 체인 연속 타격. 다수 적에게 동시 피해를 주며, 레벨에 따라 타격 횟수 증가.
☠️ 독 안개 (PoisonMist)
LV 1→3
지정 범위에 독 안개 생성. 범위 내 적에게 지속 독 피해. 레벨에 따라 데미지·지속시간 증가.
07난이도 시스템 (DifficultyManager)

Easy·Normal·Hard·Hell 4단계 난이도를 DifficultyManager가 관리합니다. 각 난이도는 적 HP·속도, 골드, 코어 HP, 웨이브 수, 중간보스 등장 웨이브를 독립적으로 조정합니다. HELL 난이도는 웨이브 1부터 중간보스가 등장해 극단적인 난이도를 제공합니다. 해금 상태는 Firebase에 동기화되어 기기를 바꿔도 유지됩니다.

C#DifficultyManager.cs
public float EnemyHpMultiplier => Selected switch
{
    Define.Difficulty.Easy   => 0.80f,
    Define.Difficulty.Normal => 1.00f,
    Define.Difficulty.Hard   => 1.30f,
    Define.Difficulty.Hell   => 1.70f,
    _ => 1f
};

public int WaveCount => Selected switch
{
    Define.Difficulty.Easy   => 10,
    Define.Difficulty.Normal => 15,
    Define.Difficulty.Hard   => 20,
    Define.Difficulty.Hell   => 25,
    _ => 10
};

public void OnGameClear()
{
    // 현재 난이도가 최대 해금일 때만 다음 단계 해금
    if (selected == data.UnlockedDifficulty && data.UnlockedDifficulty < maxIndex)
    {
        data.UnlockedDifficulty = selected + 1;
        Managers.SaveM.SaveCurrent();
    }
}
08Firebase 저장 연동

Firebase Authentication 익명 로그인 + Firestore 세이브 데이터 저장을 연동했습니다. SaveManager는 PlayerPrefs와 Firebase 이중 저장 구조이며, OnApplicationPause에서 앱 백그라운드 진입 시 자동으로 Firebase에 동기화합니다. 저장 데이터에는 스테이지 클리어 여부, 해금 난이도, 레벨/경험치, 최고 기록(스테이지·웨이브·난이도)이 포함됩니다.

09업적 · 설정 시스템

AchievementManager로 업적 달성 시 팝업 알림을 표시합니다. 설정창은 BGM/SFX 슬라이더, 그래픽 품질(Low/Medium/High) 선택을 제공하며 PlayerPrefs에 저장합니다. 배속(x1/x2)과 일시정지 팝업(계속하기/포기)도 구현되어 있으며, 일시정지 상태에서 설정 변경 후 복귀 시 timeScale이 올바르게 복구됩니다.
