🔧 기술적 구현 포인트

UI 자동 바인딩 (UI_Base)
  • Enum으로 UI 컴포넌트 이름 정의 → Bind<T>() 호출 시 FindChild로 자동 탐색 후 딕셔너리에 저장
  • GetButton(), GetText() 등으로 타입별 접근, SerializeField / Find 남용 제거
Data-Driven 설계 (Data.cs)
  • 몬스터 스탯, 강화 수치, 가챠 확률 등을 JSON 직렬화 가능한 데이터 클래스로 분리
  • ILoader<TKey, TValue> 인터페이스 + MakeDict()로 런타임에 Dictionary 변환
  • 코드 수정 없이 데이터만 교체하면 밸런스 변경 가능한 구조
커스텀 Update 루프 (ITickable)
  • MonoBehaviour.Update() 남용 방지를 위해 ITickable 인터페이스 도입
  • HashSet 기반으로 중복 등록 방지, scaled / unscaled 틱 분리 관리
  • 중앙 UpdateManager가 등록된 객체만 업데이트하여 관리 효율화
UniTask 기반 비동기 처리
  • async void 전면 제거, UniTask / UniTaskVoid로 대체
  • Firebase 데이터 로드, 씬 전환, 팝업 시퀀스 등 비동기 흐름 통일
Firebase 자동 저장 시스템
  • 10초 인터벌로 Firebase Realtime Database에 자동 저장
  • 4개 노드(DATA / CHARACTER / ITEM / SMELT)를 단일 쓰기로 묶어 원자성 보장
  • CancellationToken으로 중복 저장 요청 취소 → 항상 최신 데이터만 저장
  • 네트워크 오류 대비 최대 3회 재시도 로직
