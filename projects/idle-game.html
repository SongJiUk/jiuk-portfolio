🎮 프로젝트 개요

몬스터를 처치해 획득한 젬으로 레벨업하고, 무작위로 등장하는 스킬을 선택해 성장하는 로그라이크 방치형 RPG입니다. 구글 시트 기반 데이터 관리, 중앙 집중식 UpdateManager, Enum 기반 UI 자동 바인딩 등 실무에서 활용 가능한 패턴들을 적용했습니다.

⚙️ 핵심 시스템

01데이터 기반 콘텐츠 및 캐릭터 성장 관리

게임 밸런스 조정 및 콘텐츠 추가 시 코드 수정이 필요하다는 문제를 해결하기 위해 모든 게임 데이터를 구글 시트에서 관리하는 데이터 기반 설계를 적용했습니다. 구글 시트 → CSV → JSON 변환 파이프라인을 구축해 코드 수정 없이 밸런스와 콘텐츠를 유연하게 조정합니다.

ILoader 인터페이스를 정의하고 각 데이터 로더가 MakeDict()를 구현해 JSON 리스트를 Dictionary로 변환합니다. O(1) 조회로 런타임 성능을 최적화했습니다.

메인씬에 장비창, 캐릭터 강화, 상점, 스테이지 선택 등 다양한 탭을 구현해 플레이어가 게임 전에 충분히 준비하고 성장할 수 있는 시스템을 구축했습니다.

02핵심 게임 로직 및 ObjectPool 최적화

몬스터를 처치해 얻은 젬으로 레벨업하고, 무작위로 등장하는 스킬을 선택해 성장하는 로그라이크 플레이를 구현했습니다. 스킬 레벨이 5가 된 후 특정 아이템을 획득하면 스킬이 진화하는 시스템으로 전략적 깊이를 더했습니다.

10개의 스테이지를 구현하고, 특정 스테이지별로 엘리트 보스를 배치해 난이도를 점진적으로 상승시켰습니다. 자기장 시스템을 구현해 스테이지의 역동성과 긴장감을 높였습니다.

수백 개의 몬스터와 스킬 이펙트가 동시에 등장하는 환경을 위해 Unity ObjectPool 시스템을 구축했습니다.

03중앙 집중식 업데이트 시스템 (UpdateManager)

유니티의 개별 Update() 호출은 오브젝트 수가 많아질 때 불필요한 오버헤드를 유발합니다. ITickable 인터페이스와 UpdateManager를 구축해 모든 업데이트 로직을 일괄 관리했습니다.

04Drag & Drop 없는 Enum 기반 UI 자동 바인딩

기존의 Unity UI 수동 연결 방식은 반복 작업, 휴먼에러, 유지보수 어려움이 있었습니다. Enum과 코드 기반의 자동 UI 바인딩 시스템을 구축해 컴포넌트 간 의존성을 명확하게 관리하고 확장성을 확보했습니다.

UI_Base의 Bind<T>() 메서드가 Enum을 기반으로 컴포넌트를 찾아 자동으로 연결합니다.

BindEvent()를 통해 OnClick, OnDrag 등 모든 UI 이벤트를 한 곳에서 관리해 일관성과 재사용성을 높였습니다.
